Layout rationale

Recommendation: Layout A on desktop + Layout M on phone. Same photo set and story order. Desktop uses editorial bento bands. Mobile is its own product-scroll system — not a collapsed 3-column grid.

What the photos actually are

  • 15 studio hero shots — extreme close-ups of printed / embroidered apparel (Underdog + Forever Fierce marks).
  • Aspect mix: 10 portrait-dominant (mostly ~3:4 and ~4:5), 5 soft-landscape / near-square detail crops.
  • Job of the image: sell ink quality, type craft, fabric texture — not “people in a gym.”
  • Current /ourwork grid forces square mockups. These new heroes fight that container.

Why A wins (desktop)

  • Native ratios preserved. Portrait triptychs hold 3:4; landscape band holds the 1.1 crops; no dead letterboxing.
  • Rhythm. Big → three → three detail → color pair → bento → craft close.
  • Color pacing. Neutrals open, color breaks mid-page, embroidery closes on craft proof.

Why M exists (mobile)

  • A/B/C all collapse to the same single-column stack on a phone — correct as a fallback, weak as a concept.
  • M keeps a deliberate rhythm: full-bleed → inset → 2-up → horizontal detail rail → color bleed → alternating stack.
  • Landscape detail shots belong in a sideways rail on mobile; stacking them wastes their width.
  • Captions are always visible (hover is dead on touch).

Hard layout rules for production

  • Gap desktop: 8–12px. Mobile pair gap: 6px. Inset side padding: ~18px.
  • Corner radius: 0 on full-bleed, 4–6px on inset cards.
  • Never cover-crop a 3:4 into a square.
  • Lightbox on tap (same as current /ourwork).
  • Detail rail: scroll-snap-type: x mandatory, cards ~78% width so the next peeks.

What I would not do

  • Equal square grid (kills half the compositions).
  • Same mobile layout as collapsed desktop — ship M as the mobile stylesheet for A.
  • Heavy text between every block — one header, one closing CTA.
  • Autoplay carousels — user-owned swipe only.
